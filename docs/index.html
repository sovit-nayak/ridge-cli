How it works

Three steps. Zero friction.

Ridge CLI works with data already on your machine. No setup wizards, no accounts required.

01 ——
⚙️
Install & Start

One pip install. Ridge CLI creates a hidden folder and starts a daemon that polls every 30 seconds. Zero configuration required.

ridge start "deep work"
02 ——
👁️
Ridge CLI Watches

Reads browser history directly from disk. Reads active app via psutil. Auto-categorizes every domain. Everything stored locally in SQLite.

~/.ridge/data.db
03 ——
📊
See The Truth

Pull a terminal report anytime. Spin up a local dashboard. Get ML-powered insights on your patterns — all from your data.

ridge report

FAQ

Common questions.

Does Ridge CLI send my data anywhere?+
Never. Everything stays in ~/.ridge/data.db on your machine. Ridge CLI has zero network calls by default. The optional AI coaching feature uses the Claude API but you explicitly opt in — your raw browsing data is never sent, only processed summaries.
Do I need to install SQLite separately?+
No. SQLite is a file, not a server. Python manages it automatically the first time you run ridge start. A file called data.db appears in ~/.ridge/ — that's your entire database. No installation, no configuration, no running processes.
Which browsers does Ridge CLI support?+
Ridge CLI reads history from Chrome, Firefox, and Safari simultaneously. It accesses the SQLite history files each browser stores locally on your machine — no extension needed. Edge and Brave support is coming in v1.1.
What operating systems are supported?+
Mac, Linux, and Windows. The daemon uses platform-appropriate methods on each OS. macOS and Linux are the most stable at v1.0 — Windows support is fully functional but may have minor edge cases we're tracking.
How does the ML forecasting work?+
After 14+ days of data, Ridge CLI uses Facebook Prophet to model your personal focus rhythm. It learns your weekly cycles, identifies your peak hours, and predicts likely dip periods. All models run on your CPU via local HuggingFace models — no GPU required, no API calls made.
Is the browser extension required?+
No — Ridge CLI works without any extension by reading browser history files directly from disk. The browser extension (coming in v2.0) is an optional addition for non-developers who prefer a visual interface.
